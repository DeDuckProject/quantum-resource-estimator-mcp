# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please **do not** open a public GitHub issue.

Instead, report it by opening a [GitHub Security Advisory](../../security/advisories/new)
on this repository. This allows the issue to be assessed and patched before public disclosure.

Please include:
- A description of the vulnerability and its potential impact.
- Steps to reproduce.
- Any suggested mitigations if known.

You can expect an acknowledgement within a few days and a resolution or status update
within 14 days.
